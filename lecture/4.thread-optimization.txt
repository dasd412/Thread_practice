[일반적인 성능의 정의]
1. 지연시간: 작업을 완료하는 데 걸리는 시간
2. 처리량 : 일정 시간 동안 완료한 작업의 양

[지연 시간 줄이기]
작업 시간을 T라고 할 때, 하위 작업들을 스레드 N개로 병렬 처리한다.
일반적으로 컴퓨터의 N은 컴퓨터의 코어 개수와 최대한 비슷해야 한다. (단, 모든 스레드가 인터럽트 없이 하위 작업을 실행해야만 최적이다.)
만약 컴퓨터 코어 개수보다 스레드를 많이 할당하면 성능적으로 역효과가 난다.

[하이퍼 스레딩]
하나의 코어가 두 개의 스레드를 동시에 실행시킬 수 있는 하드웨어 기술.

[병렬처리와 그 결과를 집계하는 것의 비용]
여러 개의 작업 나누는 비용 + 스레드 생성 및 스레드에 작업을 전달하는 비용 + 스레드가 시작해서 운영체제에 스케쥴링 되는 비용 + 스레드가 끝날 때까지 신호를 기다리는 비용
+ 결과를 집약하는 비용 + 해당 결과를 저장하는 비용

[스레드 풀링]
스레드를 미리 생성 해놓고, 미래 작업을 위해 다시 스레드를 사용하는 방법.
작업들을 작업 대기열을 세워 놓은 후에, 스레드 풀에서 유휴 중인 스레드를 활용한다.
이렇게 하면 고정적인 스레드를 유지할 수 있고, 작업 할때마다 스레드를 생성해야 할 필요를 없앨 수 있다. (스레드 생성 비용 제거)

(예시)
```
int numberOfThread=4;
Executor executor=Executors.newFixedThreadPool(numberOfThread);

Runnable task=...;
executor.execute(task);
```
